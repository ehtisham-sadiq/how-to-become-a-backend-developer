# Contributing to 90-Day Backend Engineering Roadmap

First off, thank you for considering contributing to this project! It's people like you that make this roadmap a great learning resource for aspiring backend developers.

## üåü How Can I Contribute?

### Reporting Issues

If you find any issues with the roadmap, please create an issue on GitHub. Include:

- **Clear description**: What's the problem or suggestion?
- **Location**: Which day/week/section needs attention?
- **Expected vs Actual**: What did you expect vs what you found?
- **Suggestions**: How would you improve it?

### Suggesting Enhancements

We welcome suggestions for:

- Additional learning resources
- Better explanations or examples
- New technologies or tools to include
- Updated links to replace outdated resources
- Corrections to technical inaccuracies

### Contributing Content

1. **Fork the repository**
2. **Create a branch** (`git checkout -b feature/improve-week-3`)
3. **Make your changes**
4. **Test your changes** (verify links work, code examples are correct)
5. **Commit your changes** (`git commit -m 'Add improved FastAPI examples for Week 3'`)
6. **Push to your branch** (`git push origin feature/improve-week-3`)
7. **Open a Pull Request**

## üìù Content Guidelines

### Writing Style

- **Clear and concise**: Assume readers have basic Python knowledge but are new to backend
- **Structured**: Follow the existing format (Prerequisites ‚Üí Where to Learn ‚Üí Breakdown ‚Üí Exercise ‚Üí Deliverable)
- **Time-boxed**: Each day should fit within 120 minutes
- **Practical**: Focus on hands-on exercises over theory
- **Bandwidth-conscious**: Prioritize text documentation over video content

### Code Examples

- Use **Python 3.11+** syntax
- Include **type hints** wherever appropriate
- Follow **PEP 8** style guidelines
- Add **docstrings** to functions and classes
- Keep examples **simple and focused**

### Learning Resources

When adding resources:

- Prefer **official documentation** over third-party blogs
- Include **reputable sources** (Real Python, official tutorials, established YouTube educators)
- Verify **links work** before submitting
- Provide **text alternatives** when linking videos
- Consider **bandwidth limitations** of learners

## üîç Quality Standards

### Before Submitting

- [ ] All links are working and accessible
- [ ] Code examples have been tested
- [ ] Content follows the 120-minute daily structure
- [ ] Technical accuracy verified
- [ ] Spelling and grammar checked
- [ ] Consistent formatting with existing content

### Pull Request Guidelines

- **One improvement per PR**: Keep changes focused
- **Descriptive title**: "Improve Week 5 database examples"
- **Clear description**: Explain what you changed and why
- **Reference issues**: Link to related issues if applicable
